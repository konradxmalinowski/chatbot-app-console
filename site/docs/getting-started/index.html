Getting Started

Set up a virtual environment, install dependencies, configure one provider, and run your first chat. Everything runs locally except the provider's own API call.

Prerequisites

Python 3.11 or later, pip and the standard-library venv module, and credentials for one cloud model provider — or a local Ollama installation.

1 — Install dependencies

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# Install runtime dependencies
pip install -r requirements.txt

# Optional: dev tools (ruff, pytest)
pip install -r requirements-dev.txt

2 — Configure a provider

# Copy the template and fill in one provider's credentials
cp .env.example .env

The example configuration uses Gemini (GEMINI_API_KEY + GEMINI_LLM_MODEL). See Configuration & LLM Providers for the full variable table and every provider's requirements.

Before running the app, replace every bracketed placeholder in SYSTEM_PROMPT (src/constants.py) — the project name, assistant goal, supported topics, contact path, and preferred form of address. Startup validation stops the app while any placeholder remains unresolved.

3 — Run your first chat

python src/main.py

Submit an empty prompt to exit. To run fully locally instead, switch to Ollama and pull the default chat model first:

ollama pull llama3.2:3b
python src/main.py --llm-provider ollama
