  </read_first>
  <action>
    Create .git/hooks/pre-commit with the following exact content:

      #!/usr/bin/env bash
      # Pre-commit hook: block accidental .env commits.
      # Rejects any commit that attempts to add the .env file to the index.

      if git diff --cached --name-only | grep -qE "(^|/)\.env$"; then
        echo "ERROR: Attempt to commit .env detected." >&2
        echo "The .env file contains secrets and must never be committed." >&2
        echo "Remove it from staging with: git restore --staged .env" >&2
        exit 1
      fi

    After writing the file, make it executable:
      chmod +x .git/hooks/pre-commit

    Verify the hook fires correctly by testing with a dry run:
      echo "test=value" > /tmp/test_env_hook.env
      cp /tmp/test_env_hook.env .env_test_hook
      # Simulate: does the pattern match a file named ".env"?
      echo ".env" | grep -qE "(^|/)\.env$" && echo "Pattern OK" || echo "Pattern FAIL"

    The hook uses `(^|/)\.env$` to match `.env` at root or in subdirectories,
    but NOT `.env.example` or `.env.local.backup` etc.
  </action>
  <verify>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer shell → git staging | pre-commit hook intercepts git commit before objects are created |
| .env file → git objects | .env must never cross this boundary into the object store |
| requirements.txt → install | unpinned package versions allow supply-chain substitution on install |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-B | Information Disclosure | .env / GEMINI_API_KEY | mitigate | Pre-commit hook rejects any staged .env file; .env.example warning discourages workarounds |
| T-01-02-B | Tampering | requirements.txt (unpinned rich) | mitigate | Pin rich to exact version so installs are reproducible and supply-chain swaps are detectable |
| T-01-03-B | Information Disclosure | git history | mitigate | Audit via git log -p; if clean, document; if leak found, record for BFG/filter-repo remediation |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | All packages in requirements.txt are already pinned with exact versions; audit confirmed no new packages added in this plan |
</threat_model>
